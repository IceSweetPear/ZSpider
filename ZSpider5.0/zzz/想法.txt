
	Z-UI

	能不能 在 代码中  加入一个  组件
		
	这个组件的功能   跟 git 类似  就是 加载文件 

	不过 这个加载 文件   是  可控制的  实时的  在页面上 加载的

	具体是  现在的前端开发 很乱 很杂 语法很多 因为引入js css的方式 模块化 非常难以进行 而且会引入过多 无效代码
	
	因为 在 前台 上 有一些 头部 导航栏 点击按钮  Banner 图  轮播图

	模块栏  页脚   等  功能性  小模块   这些小模块 很多都是比较固定的  很多框架 也是 指出来这些模块

	所以 要把 这些 模块 真正的 模块化 

	1，模块独立化        使用 类似 laravel include 的 方式  在 主页面 上  include 每个 模块
		要点 ：这时候 css js 不再是 公共的 而是 这个 模块 私有的   虽然 取消了公共代码 看起来 会 产生 很多 多余 代码
			不过 对于小型应用 的 直接 引入 整个 框架    或是大型应用的  复杂度  这样的 方式 并不是不可行

	2，炉石传说化       单个单个的模块元素 虽然 没有公共的 样式   不过 可以加入 公共样式 的功能
		要点： 单个实体元素 具有（独立样式）  类似 （随从）   
			也可以加入 （群体法术） 也就是 （公共样式）可以控制指定范围元素颜色 指定范围元素 大小之类的
				进而可以 快速 批量的 改变元素 而不用 重复操作
				因为 群体法术 改变元素 大小 颜色 等 特殊性  必须 严格 统一 元素的 样式
			也可以加入 （单体法术）单体法术 主要控制 指定元素 的 跳转 点击行为  等针对单体 元素的 操作
		
	3， 拖拽布局化      在核心功能上 有个 可以 拖动 元素的 功能   因为 模块化 的时候 可能 会增加 很多模块  超多模块 要是全记下来
			也是一个不小的 负担 所以有一个页面  就负责  展示所有模块  类似 QQ分组的模式  
			默认带有系统自带的组件  
			而拖拽布局 可以非常简易的 完成 代码布局操作 而且  用户 可以div自己的布局策略

	4， 全面rem定宽化	   由于适配问题  前端 要适配各种端屏幕大小 或使用 框架的栅格化 但是这种适配 虽然 效果不错 但是 极其复杂 难以扩展
			   Z-UI  不做 复杂的 操作  所以 设定为 ：  主要适配 pc 端 16：9 这种布局 全面使用 rem布局操作
			  让 html font size 跟着 屏幕宽度改变  
			  使得大屏幕 元素更大 小屏幕 元素更小  超小屏幕就别看了。。
			  定宽不定高 ：整体页面 不得 左右滑动  只可 上下滑动    
			  手机旋转： 由于采用 16：9的原因 手机端 默认 横向 观看   竖向 元素 特别小

	5， 前端框架和后台框架   由于拖动啊收藏夹分组啊之类的操作 单前端完成不了 所以 分为前端框架 和 后台框架
		前端框架：模块化 定制模块操作 为自己写代码 完成命名 自己在承载页面中 引入进去 完成模块化
		后台框架：增加拖拽布局 收藏夹 分组 等功能
